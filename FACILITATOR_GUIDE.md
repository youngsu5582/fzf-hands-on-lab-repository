## 30분 타임라인 상세

### 0-5분: 인트로 & 환경 설정

**목표:** 참가자들의 관심 유도 및 환경 확인

**진행:**
1. **자기 소개 및 워크샵 소개** (1분)
   - "안녕하세요! 오늘은 CLI 생산성을 10배 높여주는 fzf를 배워보겠습니다."
   - 워크샵 목표와 흐름 간단히 소개

2. **fzf 데모** (2분)
   - `history | fzf` 실행하여 wow 효과 만들기
   - `find . | fzf --preview='cat {}'` 실행
   - "이런 것들을 30분 안에 여러분도 만들 수 있습니다!"

3. **환경 확인** (2분)
   ```bash
   # 참가자들에게 실행하도록 안내
   fzf --version
   ```
   - 설치 안 된 사람: Docker 환경 안내 또는 빠른 설치
   - Step 0 README 참고하도록 안내

**발표 팁:**
- 에너지 넘치게 시작하기
- 간단하지만 임팩트 있는 데모로 호기심 자극
- 환경 문제는 빠르게 처리 (미리 준비한 Docker 환경 활용)

### 5-12분: Step 1 - 기본 사용법

**목표:** fzf의 기본 개념과 검색 패턴 익히기

**진행:**
1. **기본 사용법 시연** (2분)
   ```bash
   cd step/step1
   cat sample.txt | fzf
   ```
   - 화살표 키로 이동
   - 검색어 입력하며 실시간 필터링 보여주기
   - Enter로 선택, ESC로 취소

2. **검색 패턴 실습** (3분)
   - Fuzzy 검색: `music`
   - 정확 검색: `'music`
   - 끝 패턴: `.mp3$`
   - 제외: `!music`
   - AND 조건: `music mp4`

   **참가자 따라하기 시간 1분 포함**

3. **problem.zsh 함수 실습** (2분)
   ```bash
   source problem.zsh
   find-sample
   ```
   - 함수 구조 간단히 설명
   - "이렇게 쉘 함수로 만들어 재사용할 수 있습니다"

**발표 팁:**
- 각 패턴을 시연할 때 "이런 경우에 유용합니다" 설명
- 참가자들이 따라할 시간 충분히 주기
- 막히는 사람 있으면 빠르게 도와주기

**예상 질문:**
- Q: "정규표현식 다 지원하나요?"
  - A: "네, ripgrep 스타일 정규표현식을 지원합니다. `man fzf`에서 확인하세요."

---

### 12-20분: Step 2 - 고급 옵션

**목표:** preview, query 등 주요 옵션 활용법 익히기

**진행:**
1. **preview 옵션** (3분)
   ```bash
   cd ../step2
   find . | fzf --preview='cat {1}'
   find . | fzf --preview='ls -al {1}'
   find . | fzf --preview-window=right:50% --preview='stat {1}'
   ```
   - "실시간으로 파일 내용을 미리 볼 수 있습니다"
   - preview-window로 위치/크기 조절 가능

2. **query 옵션** (2분)
   ```bash
   find . | fzf --query='music' --preview='cat {1}'
   ```
   - "자주 검색하는 키워드를 기본값으로 설정"

3. **delimiter & with-nth** (2분)
   ```bash
   cat endpoints.txt | fzf --with-nth=1,3 --delimiter=$'\t' \
       --header="ENDPOINT | HANDLER"
   ```
   - "TSV, CSV 같은 구조화된 데이터 처리에 유용"

4. **problem.zsh 버그 찾기 챌린지** (1분)
   ```bash
   source problem.zsh  # 에러 발생
   ```
   - "버그가 있습니다! 찾아서 수정해보세요"
   - 참가자들에게 1분 시간 제공
   - 답: `local filePath = "./folder"` → `local filePath="./folder"` (공백 제거)

**발표 팁:**
- 옵션이 많으므로 핵심 3-4개만 집중
- 실용적인 use case와 함께 설명
- "여러분의 업무에서는 어떻게 활용할 수 있을까요?" 질문 던지기

**예상 질문:**
- Q: "preview에서 이미지도 볼 수 있나요?"
  - A: "터미널에 따라 다릅니다. kitty, iTerm2 등에서는 가능합니다."

### 20-28분: Step 3 - 실전 활용

**목표:** 실무에서 바로 쓸 수 있는 파이프라인 구축

**진행:**
1. **파이프라인 개념 설명** (1분)
   ```
   INPUT | fzf | OUTPUT
   ```
   - "fzf는 중간 다리 역할"
   - "다양한 CLI 도구와 결합 가능"

2. **Spring Boot Actuator 예제** (4분)
   ```bash
   cd ../step3
   cat sample.json
   cat sample.json | jq -r '.loggers | keys[]'
   ```
   - jq로 데이터 추출하는 방법 시연
   - answer.zsh의 spring-loggers 함수 구조 설명
   - (서버가 있다면) 실제 실행 데모

3. **Git 활용 예제** (2분)
   ```bash
   source git-answer.zsh
   # git 저장소에서 실행
   fzf-git-checkout
   fzf-git-log
   ```
   - "브랜치 전환, 커밋 검색이 이렇게 편해집니다"

4. **자유 질문 & 추가 예제** (1분)
   - 프로세스 kill 예제
   - 디렉토리 이동 예제
   - 참가자들의 use case 청취

**발표 팁:**
- 실전 예제는 "와, 이거 바로 써야겠다" 반응 유도
- Git 예제는 개발자 청중에게 특히 효과적
- 시간 여유 있으면 참가자 요청 받아 즉석 함수 작성

**예상 질문:**
- Q: "AWS CLI와도 결합할 수 있나요?"
  - A: "네! Step 3 README에 EC2 인스턴스 선택 예제가 있습니다."

## 자주 하는 실수 & 대처법

### 실수 1: 시간 배분 실패
**문제:** Step 1-2에서 시간을 너무 써서 Step 3을 못 함
**해결:**
- Step 1은 7분 넘기지 않기 (타이머 설정)
- Step 2에서 옵션 테이블은 빠르게 넘어가기
- Step 3이 가장 임팩트 있으므로 시간 확보 필수

### 실수 2: 너무 많은 옵션 설명
**문제:** 모든 옵션을 다 설명하려다 지루해짐
**해결:**
- 핵심 3-4개만 집중 (preview, query, delimiter, layout)
- "더 많은 옵션은 `man fzf` 참고하세요"
- 실용성 위주로 설명

### 실수 3: 데모 실패
**문제:** 라이브 코딩 중 에러 발생
**해결:**
- 사전에 모든 명령어 테스트
- 명령어 히스토리에 미리 저장해두기
- 에러 나면 당황하지 말고 "실전에서는 이런 일도 있죠" 여유 있게

### 실수 4: 참가자 레벨 미스매치
**문제:** 초보자와 고급자가 섞여 있음
**해결:**
- 초반에 "CLI 익숙한 분?" 손 들어보라고 하기
- 고급자에겐 "answer.zsh 먼저 보고 응용해보세요"
- 초보자에겐 "천천히 따라하세요, 돕겠습니다"

## 환경별 트러블슈팅

### Mac
```bash
# Homebrew로 설치
brew install fzf

# 쉘 통합 스크립트 실행 (선택)
$(brew --prefix)/opt/fzf/install
```

### Windows WSL
```bash
sudo apt-get update
sudo apt-get install -y fzf
```

### Docker (최후의 수단)
```bash
docker-compose up -d
docker exec -it fzf_shell zsh
```
